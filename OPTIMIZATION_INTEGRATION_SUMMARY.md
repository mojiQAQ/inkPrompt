### 设计理念
- **位置合理**：靠近内容输入框，方便快速访问
- **视觉突出**：紫色渐变设计，与 Token 计数形成视觉平衡
- **条件显示**：仅在编辑模式显示，避免新建时困惑
- **反馈清晰**：优化中显示加载状态，完成后 Toast 提示

## 🔄 工作流程

### 用户操作流程
```
1. 打开提示词编辑页面（必须是已保存的提示词）
   ↓
2. 找到内容输入框右上角的 "AI 优化" 按钮
   ↓
3. 点击按钮，在下拉菜单中选择优化场景
   ↓
4. 等待 3-10 秒，系统调用 GPT 进行优化
   ↓
5. 优化完成，内容自动更新到输入框
   ↓
6. 查看成功提示（显示 Token 和成本）
   ↓
7. 保存更改或撤销（Ctrl+Z / Cmd+Z）
```

### 技术流程
```
用户点击场景
   ↓
OptimizeButton.handleOptimize()
   ↓
调用 API: POST /api/prompts/{id}/optimize
   ↓
后端 LangChain 处理（optimizationService）
   ↓
GPT 模型返回优化结果
   ↓
返回给前端：optimized_content, token_count, cost
   ↓
调用 onOptimized(optimizedContent)
   ↓
PromptEditor.handleOptimized()
   ↓
setContent(optimizedContent) - 更新输入框
   ↓
toast.success() - 显示成功提示
```

## 🔍 技术细节

### 组件通信
- **Props 传递**：
  - `promptId`：当前提示词的 ID
  - `onOptimized`：优化完成后的回调函数

- **回调设计**：
  ```typescript
  const handleOptimized = (optimizedContent: string) => {
    setContent(optimizedContent)  // 更新内容
    toast.success('内容已更新为优化后的版本')  // 用户反馈
  }
  ```

### 状态管理
- **本地状态**：使用 React useState 管理内容
- **无需全局状态**：优化是即时操作，不涉及全局状态
- **版本控制**：保存时会自动创建新版本，无需特殊处理

### 错误处理
- **前端层面**：OptimizeButton 内部处理 API 错误，显示错误 Toast
- **用户操作**：如果优化失败，用户可以重试或手动编辑
- **数据安全**：优化不会自动保存，用户可以撤销或取消

## 💰 成本控制

### 透明度
- 每次优化后显示实际消耗的 Token 数量
- 显示估算的 API 成本（美元）
- 用户在点击前就知道会产生成本

### 成本范围
- 短文本（< 200 字）：$0.0001
- 中等长度（200-500 字）：$0.0003
- 长文本（500-1000 字）：$0.0007
- 超长文本（> 1000 字）：$0.0015+

### 优化建议
- 文档中提供节省成本的建议
- 建议精简内容后再优化
- 避免重复优化同一内容
