# 提示词管理规范

## ADDED Requirements

### Requirement: 创建提示词
系统 SHALL 允许用户创建新的提示词记录,包含名称、内容和标签信息。

#### Scenario: 成功创建提示词
- **WHEN** 用户填写提示词名称和内容后点击保存
- **THEN** 系统应验证名称不为空
- **AND** 系统应验证内容不为空
- **AND** 创建新的提示词记录
- **AND** 自动创建初始版本记录 (version 1)
- **AND** 自动计算并存储 Token 数量
- **AND** 关联用户选择的标签
- **AND** 返回创建成功的提示词详情
- **AND** 显示成功提示消息

#### Scenario: 名称为空时创建失败
- **WHEN** 用户未填写提示词名称就点击保存
- **THEN** 系统应显示 "提示词名称不能为空" 错误提示
- **AND** 不执行创建操作
- **AND** 焦点定位到名称输入框

#### Scenario: 内容为空时创建失败
- **WHEN** 用户未填写提示词内容就点击保存
- **THEN** 系统应显示 "提示词内容不能为空" 错误提示
- **AND** 不执行创建操作
- **AND** 焦点定位到内容输入框

#### Scenario: 创建时关联标签
- **WHEN** 用户在创建提示词时选择了多个标签
- **THEN** 系统应创建提示词与标签的关联关系
- **AND** 在提示词详情中显示所有关联标签
- **AND** 如果标签不存在,先创建标签再关联

### Requirement: 查询提示词列表
系统 SHALL 提供提示词列表查询功能,支持分页、搜索和筛选。

#### Scenario: 查询用户所有提示词
- **WHEN** 用户访问提示词列表页面
- **THEN** 系统应返回当前用户的所有提示词
- **AND** 按更新时间倒序排列
- **AND** 每页显示最多 50 条记录
- **AND** 提供分页导航

#### Scenario: 搜索提示词
- **WHEN** 用户在搜索框输入关键词
- **THEN** 系统应搜索提示词名称和内容
- **AND** 返回包含关键词的提示词列表
- **AND** 高亮显示匹配的关键词
- **AND** 显示搜索结果数量

#### Scenario: 按标签筛选提示词
- **WHEN** 用户选择一个或多个标签进行筛选
- **THEN** 系统应返回包含所选标签的提示词
- **AND** 多个标签之间为 OR 关系 (包含任一标签即可)
- **AND** 在列表上方显示当前筛选条件

#### Scenario: 空列表状态
- **WHEN** 用户没有任何提示词
- **THEN** 系统应显示空状态提示
- **AND** 提供 "创建第一个提示词" 的引导按钮
- **AND** 显示产品使用说明

### Requirement: 查看提示词详情
系统 SHALL 允许用户查看单个提示词的完整信息。

#### Scenario: 查看提示词详情
- **WHEN** 用户点击提示词卡片
- **THEN** 系统应显示提示词的完整内容
- **AND** 显示提示词名称、内容、创建时间、更新时间
- **AND** 显示关联的所有标签
- **AND** 显示 Token 统计信息
- **AND** 提供编辑和删除操作按钮

#### Scenario: 查看不存在的提示词
- **WHEN** 用户尝试访问不存在的提示词 ID
- **THEN** 系统应返回 404 错误
- **AND** 显示 "提示词不存在" 错误提示
- **AND** 提供返回列表的按钮

#### Scenario: 查看其他用户的提示词
- **WHEN** 用户尝试访问其他用户的提示词
- **THEN** 系统应返回 403 Forbidden 错误
- **AND** 显示 "无权访问该提示词" 错误提示

### Requirement: 更新提示词
系统 SHALL 允许用户修改提示词的名称、内容和标签,内容修改时自动创建新版本。

#### Scenario: 更新提示词内容
- **WHEN** 用户修改提示词内容并保存
- **THEN** 系统应更新提示词记录
- **AND** 创建新的版本记录 (version + 1)
- **AND** 重新计算并更新 Token 数量
- **AND** 更新提示词的更新时间
- **AND** 显示更新成功提示

#### Scenario: 更新提示词标签
- **WHEN** 用户修改提示词的标签
- **THEN** 系统应更新标签关联关系
- **AND** 不创建新的版本记录 (仅标签变化不产生版本)
- **AND** 更新提示词的更新时间
- **AND** 显示更新成功提示

#### Scenario: 同时更新内容和标签
- **WHEN** 用户同时修改提示词内容和标签
- **THEN** 系统应创建新的版本记录 (因为内容变化)
- **AND** 更新标签关联关系
- **AND** 重新计算 Token 数量
- **AND** 更新提示词的更新时间

#### Scenario: 更新为空内容失败
- **WHEN** 用户将提示词内容清空后保存
- **THEN** 系统应显示 "提示词内容不能为空" 错误提示
- **AND** 不执行更新操作
- **AND** 保持原有内容不变

### Requirement: 删除提示词
系统 SHALL 允许用户删除提示词,删除前需要用户确认。

#### Scenario: 删除提示词前确认
- **WHEN** 用户点击删除按钮
- **THEN** 系统应显示确认对话框
- **AND** 对话框应显示 "确定要删除该提示词吗?此操作不可恢复"
- **AND** 提供 "取消" 和 "确认删除" 按钮

#### Scenario: 确认删除提示词
- **WHEN** 用户在确认对话框中点击 "确认删除"
- **THEN** 系统应删除提示词记录
- **AND** 删除所有关联的版本记录
- **AND** 删除所有标签关联关系
- **AND** 返回提示词列表页面
- **AND** 显示 "提示词已删除" 成功提示

#### Scenario: 取消删除
- **WHEN** 用户在确认对话框中点击 "取消"
- **THEN** 系统应关闭对话框
- **AND** 不执行删除操作
- **AND** 停留在当前页面

### Requirement: 提示词列表展示
系统 SHALL 在列表中以卡片形式展示提示词的关键信息。

#### Scenario: 提示词卡片展示
- **WHEN** 用户查看提示词列表
- **THEN** 每个提示词卡片应显示以下信息:
  - 提示词名称 (完整显示)
  - 提示词内容 (最多显示前 100 个字符,超出显示省略号)
  - 所有关联标签 (标签徽章样式)
  - Token 统计 (如: "约 1,234 tokens")
  - 更新时间 (相对时间,如 "2 小时前")
- **AND** 卡片应支持点击进入详情
- **AND** 卡片应提供快捷操作按钮 (编辑、删除)

#### Scenario: 长内容缩略显示
- **WHEN** 提示词内容超过 100 个字符
- **THEN** 卡片中应显示前 100 个字符
- **AND** 末尾显示 "..." 省略号
- **AND** 点击卡片可查看完整内容

### Requirement: 提示词版本记录
系统 SHALL 自动记录提示词内容的每次修改,创建版本历史。

#### Scenario: 创建初始版本
- **WHEN** 用户创建新的提示词
- **THEN** 系统应创建版本号为 1 的版本记录
- **AND** 版本记录包含: 版本号、内容、Token 数、修改时间
- **AND** 关联到对应的提示词

#### Scenario: 更新内容创建新版本
- **WHEN** 用户修改提示词内容
- **THEN** 系统应创建新的版本记录
- **AND** 版本号自动递增 (当前最大版本号 + 1)
- **AND** 保存新的内容和 Token 数
- **AND** 记录修改时间

#### Scenario: 版本号连续性
- **WHEN** 系统创建新版本
- **THEN** 版本号应保持连续递增
- **AND** 不允许跳号或重复
- **AND** 版本号从 1 开始计数

### Requirement: 分页功能
系统 SHALL 实现提示词列表的分页功能,优化大数据量展示。

#### Scenario: 分页参数
- **WHEN** 前端请求提示词列表
- **THEN** 系统应支持 page 和 page_size 参数
- **AND** page 默认为 1
- **AND** page_size 默认为 50,最大为 100
- **AND** 返回总记录数、总页数、当前页码等分页信息

#### Scenario: 分页导航
- **WHEN** 用户查看提示词列表
- **THEN** 系统应显示分页导航组件
- **AND** 显示当前页码和总页数
- **AND** 提供上一页、下一页按钮
- **AND** 提供页码跳转功能
- **AND** 首页时禁用上一页按钮
- **AND** 末页时禁用下一页按钮

### Requirement: 加载状态和错误处理
系统 SHALL 提供清晰的加载状态和错误提示,提升用户体验。

#### Scenario: 加载状态显示
- **WHEN** 系统正在加载提示词列表或详情
- **THEN** 应显示加载指示器 (Spinner 或骨架屏)
- **AND** 禁用所有交互操作
- **AND** 显示 "加载中..." 文本提示

#### Scenario: 网络错误处理
- **WHEN** API 请求因网络问题失败
- **THEN** 系统应显示 "网络连接失败,请稍后重试" 错误提示
- **AND** 提供重试按钮
- **AND** 记录错误日志

#### Scenario: 服务器错误处理
- **WHEN** 后端返回 500 错误
- **THEN** 系统应显示 "服务器错误,请稍后重试" 错误提示
- **AND** 提供刷新页面按钮
- **AND** 记录详细错误日志
