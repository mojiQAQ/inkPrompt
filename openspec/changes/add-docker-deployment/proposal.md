# Docker 容器化部署提案

## Why

当前 Ink & Prompt 项目的部署需要手动配置前后端环境，包括 Node.js、Python 虚拟环境、数据库迁移等多个步骤，给开发、测试和生产环境的部署带来了较高的复杂度和出错风险。

为了实现一键部署、环境一致性和快速扩展，需要将项目容器化：
- **简化部署流程**：通过 Docker Compose 实现一键启动所有服务
- **环境一致性**：消除"在我机器上能跑"的问题
- **便于扩展**：为后续 CI/CD、云部署、多环境管理打下基础
- **依赖隔离**：确保所有依赖都在容器内管理，避免环境污染

## What Changes

### 1. 后端 Dockerfile
- 基于 Python 3.11 官方镜像
- 使用多阶段构建优化镜像大小
- 配置虚拟环境和依赖安装
- 暴露 8000 端口（FastAPI 服务）
- 设置健康检查

### 2. 前端 Dockerfile
- 基于 Node.js 18 官方镜像的多阶段构建
- 第一阶段：构建生产资源
- 第二阶段：使用 Nginx 服务静态文件
- 暴露 80 端口（Nginx）
- 配置 Nginx 代理转发到后端 API

### 3. Docker Compose 配置
- 定义 frontend、backend、database 三个服务
- 配置服务间网络连接
- 设置环境变量管理
- 配置数据持久化（数据库卷）
- 设置服务依赖关系和启动顺序

### 4. 环境配置文件
- `.env.example` 模板文件
- 区分开发和生产环境配置
- 敏感信息（API Key）通过环境变量注入

### 5. Nginx 配置
- SPA 路由支持（所有路由回退到 index.html）
- API 代理配置（`/api` 转发到后端）
- 静态资源缓存策略
- Gzip 压缩

### 6. 部署文档
- Docker 安装指南
- 快速启动命令
- 常见问题排查
- 环境变量说明

## Impact

### 受影响的规范
- **NEW**: `docker-deployment` - Docker 容器化部署规范

### 新增文件
- `backend/Dockerfile` - 后端容器镜像定义
- `frontend/Dockerfile` - 前端容器镜像定义
- `frontend/nginx.conf` - Nginx 配置文件
- `docker-compose.yml` - 服务编排配置
- `docker-compose.dev.yml` - 开发环境配置（可选）
- `.env.example` - 环境变量模板
- `.dockerignore` - Docker 构建忽略文件（前后端各一个）
- `docs/DEPLOYMENT.md` - 部署文档

### 受影响的现有文件
- `backend/.env` - 添加到 .gitignore（如果尚未添加）
- `README.md` - 添加 Docker 部署说明章节

### 技术风险
- **网络配置**：容器间网络需要正确配置，否则前端无法访问后端 API
- **数据持久化**：SQLite 数据库文件需要通过卷挂载持久化
- **环境变量**：敏感信息（API Key）需要妥善管理，避免泄漏
- **镜像大小**：需要优化镜像大小，避免构建时间过长

### 业务影响
- **开发效率**：开发者可以快速启动完整环境，无需手动配置
- **部署简化**：生产环境部署只需一条命令
- **环境隔离**：多个开发者可以独立运行项目，互不干扰

### 非功能性影响
- **可移植性**：项目可以在任何支持 Docker 的平台上运行
- **可维护性**：依赖关系明确，版本锁定
- **可扩展性**：为后续容器编排（Kubernetes）做准备
