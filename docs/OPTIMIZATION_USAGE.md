# 提示词优化功能使用指南

## 📖 功能概述

InkPrompt 的提示词优化功能是一个强大的 AI 辅助工具，可以帮助你改进提示词的质量、清晰度和效果。通过 LangChain 和 OpenAI GPT 模型的集成，系统会根据不同的使用场景为你的提示词提供专业的优化建议。

## 🎯 核心优势

- **智能优化**：基于 GPT-4 的深度语言理解，提供专业的改进建议
- **多场景支持**：针对 5 种不同场景定制优化策略
- **成本透明**：实时显示 Token 消耗和成本估算
- **无缝集成**：直接在编辑器中一键优化，结果立即应用
- **版本控制**：优化后自动创建新版本，可随时回滚

## 🚀 如何使用

### 1. 进入编辑模式

优化功能仅在**编辑已有提示词**时可用。你需要：

1. 在提示词列表页面选择一个提示词
2. 点击卡片进入编辑模式
3. 或者点击提示词卡片上的"编辑"按钮

> ⚠️ **注意**：新建提示词时无法使用优化功能。你需要先保存提示词，然后再编辑它来使用优化功能。

### 2. 找到优化按钮

在编辑器页面，你会在**提示词内容输入框的右上角**看到一个紫色渐变的 **"AI 优化"** 按钮，它位于 Token 计数显示的右侧。

```
┌──────────────────────────────────────────────────────┐
│ 提示词内容 *              📊 250 tokens  [AI 优化 ⚡] │
├──────────────────────────────────────────────────────┤
│                                                      │
│  [提示词内容输入框]                                   │
│                                                      │
└──────────────────────────────────────────────────────┘
```

### 3. 选择优化场景

点击 **"AI 优化"** 按钮后，会弹出一个下拉菜单，显示 5 种优化场景：

#### ✨ 通用优化
- **适用于**：各类提示词的通用改进
- **优化方向**：
  - 提高清晰度和可读性
  - 优化结构和逻辑
  - 增强表达的准确性
  - 减少歧义和冗余
- **示例用途**：日常使用的各类提示词

#### ✍️ 内容创作
- **适用于**：写作、文章生成、创意内容
- **优化方向**：
  - 增强创意表达
  - 丰富语言描述
  - 优化叙事结构
  - 提升内容吸引力
- **示例用途**：博客文章、营销文案、故事创作

#### 💻 代码生成
- **适用于**：编程、算法、技术实现
- **优化方向**：
  - 明确技术需求
  - 规范化术语使用
  - 补充技术细节
  - 优化代码风格说明
- **示例用途**：代码生成、API 调用、算法实现

#### 📊 数据分析
- **适用于**：数据处理、统计分析、报表生成
- **优化方向**：
  - 明确数据要求
  - 规范化指标定义
  - 优化分析逻辑
  - 增强可解释性
- **示例用途**：数据可视化、趋势分析、报告生成

#### 💬 对话交互
- **适用于**：聊天机器人、客服、角色扮演
- **优化方向**：
  - 优化对话流程
  - 增强互动性
  - 明确角色定位
  - 改进回复风格
- **示例用途**：智能客服、虚拟助手、角色对话

### 4. 等待优化完成

选择场景后：

1. 按钮会显示 **"优化中..."** 状态
2. 系统会调用 GPT 模型进行分析和优化
3. 通常需要 3-10 秒，具体取决于提示词长度

### 5. 查看优化结果

优化完成后：

1. **成功提示**：屏幕右上角会弹出绿色提示框
   ```
   ✅ 优化完成！
   Token: 280 | 成本: $0.0012
   ```

2. **内容更新**：优化后的内容会**自动替换**输入框中的原内容

3. **版本创建**：保存时会自动创建新版本，更新说明中会包含"AI 优化"标记

### 6. 保存或回滚

优化后，你有两个选择：

#### 选项 A：保存优化结果
1. 点击底部的 **"保存更改"** 按钮
2. 系统会创建一个新版本
3. 更新说明中会自动记录优化信息

#### 选项 B：撤销优化
1. 使用浏览器的撤销功能（Ctrl+Z 或 Cmd+Z）
2. 或者点击 **"取消"** 按钮离开编辑页面
3. 也可以从版本历史中恢复之前的版本

## 💰 成本说明

### Token 计算
- **输入 Token**：原提示词内容 + 优化指令
- **输出 Token**：优化后的提示词 + 改进建议
- **估算公式**：输入 Token ≈ 原内容字符数 / 4

### 定价示例

使用 GPT-3.5-turbo 模型（默认）：

| 提示词长度 | 估算 Token | 输入成本 | 输出成本 | 总成本 |
|-----------|-----------|----------|----------|--------|
| 100 字符  | ~25       | $0.00001 | $0.00004 | $0.00005 |
| 500 字符  | ~125      | $0.00006 | $0.00019 | $0.00025 |
| 1000 字符 | ~250      | $0.00013 | $0.00038 | $0.00051 |

> 💡 **提示**：大多数优化操作的成本在 $0.0001 - $0.0010 之间

### 节省成本的建议
1. **精简内容**：优化前适当删减冗余内容
2. **选择合适场景**：准确的场景选择可提高优化效率
3. **避免重复优化**：对于已优化的提示词，避免反复优化
4. **批量处理**：集中优化多个提示词，减少切换开销

## 🎓 最佳实践

### 1. 优化前的准备

**明确优化目标**
- 你希望改进什么方面？（清晰度、专业性、创意性）
- 目标用户是谁？（技术人员、普通用户、特定领域）
- 期望的输出风格？（正式、随意、简洁、详细）

**清理原内容**
- 删除明显的错误和冗余
- 保留核心意图和关键信息
- 确保基本的语法正确性

### 2. 选择合适的场景

| 如果你的提示词... | 选择场景 |
|----------------|---------|
| 用于生成代码或技术文档 | 💻 代码生成 |
| 用于写文章、营销文案 | ✍️ 内容创作 |
| 用于数据处理和分析 | 📊 数据分析 |
| 用于聊天机器人或客服 | 💬 对话交互 |
| 不确定或混合用途 | ✨ 通用优化 |

### 3. 审查优化结果

优化后，请仔细检查：

**✅ 检查清单**
- [ ] 核心意图是否保留？
- [ ] 新内容是否更清晰？
- [ ] 是否有意外的内容变化？
- [ ] 专业术语是否准确？
- [ ] 整体结构是否改进？

**常见优化改进**
- ✅ 结构化：添加编号、分点说明
- ✅ 明确化：补充上下文和约束条件
- ✅ 精炼化：删除冗余表达
- ✅ 专业化：使用领域专业术语

### 4. 迭代优化

对于复杂提示词，建议采用**渐进式优化**：

```
原始提示词
    ↓ 第一次优化（通用优化）
基础优化版本
    ↓ 手动调整
手动调整版本
    ↓ 第二次优化（场景特定）
最终优化版本
```

## 🔍 常见问题

### Q1: 为什么新建提示词时看不到优化按钮？

**A**: 优化功能需要提示词的 ID 来调用 API。新建时提示词还未保存到数据库，因此没有 ID。

**解决方案**：
1. 先填写基本信息（名称、内容）
2. 点击"创建提示词"保存
3. 在列表中找到刚创建的提示词
4. 点击进入编辑模式，此时就能使用优化功能了

### Q2: 优化失败怎么办？

**可能原因**：
- ❌ OpenAI API key 未配置或失效
- ❌ 网络连接问题
- ❌ 提示词内容为空
- ❌ API 配额已用尽

**解决方案**：
1. 检查后端日志确认具体错误
2. 验证 `.env` 文件中的 `OPENAI_API_KEY` 配置
3. 确保网络可以访问 OpenAI API
4. 检查 OpenAI 账户配额和余额

### Q3: 优化后的内容不满意怎么办？

**方案 1：立即撤销**
- 使用 Ctrl+Z (Windows) 或 Cmd+Z (Mac) 撤销

**方案 2：重新优化**
- 选择不同的优化场景再试一次
- 或者手动编辑后再优化

**方案 3：版本回滚**
- 点击页面下方的"版本历史"
- 找到优化前的版本
- 点击"恢复到此版本"

### Q4: 可以连续优化多次吗？

**可以，但不建议。**

- ✅ **推荐**：优化 1-2 次，然后手动调整
- ⚠️ **不建议**：连续优化 3 次以上

**原因**：
1. 多次优化可能偏离原始意图
2. 成本会累积增加
3. 可能产生过度优化的内容

**建议流程**：
```
原始 → 优化 → 手动调整 → 完成
```

### Q5: 优化会修改提示词的核心逻辑吗？

**不会。** 优化算法的目标是：
- ✅ 改进表达方式
- ✅ 优化结构组织
- ✅ 增强清晰度
- ❌ **不会**改变核心意图
- ❌ **不会**删除关键信息

但为了安全起见，建议：
1. 优化前先复制原内容做备份
2. 优化后仔细审查结果
3. 利用版本控制功能

### Q6: 支持哪些语言？

**支持所有语言，包括：**
- ✅ 中文（简体、繁体）
- ✅ 英文
- ✅ 日文、韩文
- ✅ 其他主流语言

优化会自动识别输入语言，并使用相同语言输出结果。

### Q7: 优化需要多长时间？

**通常 3-10 秒**，具体取决于：

| 因素 | 影响 |
|-----|------|
| 提示词长度 | 越长处理时间越久 |
| OpenAI API 响应速度 | 高峰期可能较慢 |
| 网络状况 | 影响 API 调用速度 |
| 选择的模型 | GPT-4 比 GPT-3.5 慢 |

> 💡 **提示**：如果超过 30 秒仍未完成，可能出现了网络或 API 问题

## 📊 优化效果示例

### 示例 1：通用优化

**优化前**：
```
帮我写一篇关于人工智能的文章要包括AI的定义发展历史和未来趋势内容要详细一点
```

**优化后**：
```
请创作一篇关于人工智能的综合性文章，需要涵盖以下三个主要部分：

1. **AI 的定义**：清晰阐述人工智能的核心概念和基本原理
2. **发展历史**：梳理 AI 从诞生到现在的重要里程碑和技术演进
3. **未来趋势**：分析和预测 AI 技术的发展方向和潜在影响

要求：
- 内容详实，每个部分不少于 300 字
- 语言通俗易懂，适合非技术读者
- 引用具体案例和数据支撑观点
```

**改进点**：
- ✅ 结构化为 3 个清晰的部分
- ✅ 添加了具体的格式要求
- ✅ 明确了目标受众和风格

### 示例 2：代码生成优化

**优化前**：
```
写一个 Python 函数实现冒泡排序
```

**优化后**：
```
请使用 Python 编写一个冒泡排序函数，具体要求如下：

**函数规格**：
- 函数名：`bubble_sort`
- 参数：`arr: list[int]` - 待排序的整数列表
- 返回值：`list[int]` - 排序后的列表（升序）

**实现要求**：
1. 使用标准的冒泡排序算法
2. 添加详细的代码注释
3. 包含函数文档字符串（docstring）
4. 提供 2-3 个测试用例

**性能考虑**：
- 时间复杂度：O(n²)
- 空间复杂度：O(1)
- 原地排序，不创建新列表
```

**改进点**：
- ✅ 明确函数签名和类型提示
- ✅ 详细列出实现要求
- ✅ 补充性能指标说明
