# 落地页设计方案

## 概述

为 Ink & Prompt 设计一个现代化、吸引人的产品落地页，帮助用户快速了解产品核心功能并开始使用。

### 设计目标

- **目标用户**：AI 提示词创作者、内容创作者、开发者、设计师
- **核心价值**：提供完整的提示词管理、优化、测试一体化解决方案
- **转化目标**：引导新用户注册并创建第一个提示词

## 导航栏设计

### 产品级导航结构

落地页需要展示产品级别的导航，而非仅仅是页面内的锚点跳转。

| 元素 | 功能 | 跳转目标 |
|------|------|-----------|
| Logo | 返回首页 | `/` 或 `/landing` 刷新页面 |
| 提示词管理 | 已登录用户快速入口 | `/prompts` |
| 文档 | 了解详细使用方法 | `/docs` 或外部文档链接 |
| 语言切换 | 切换页面语言 | 刷新当前页并应用新语言 |
| 登录/注册 | 用户认证入口 | 点击后打开登录对话框或跳转 `/login` |

### 导航栏布局

```
┌──────────────────────────────────────────────────────────────────────────────┐
│                                                                    │
│  ┌─────┐   提示词管理    文档        🌐 中文 ▼        [登录]  │
```

## 用户旅程

### 新用户首次访问流程

```
用户访问落地页
      ↓
浏览 Hero 区域 → 了解核心价值
      ↓
查看功能卡片 → 具体功能介绍
      ↓
阅读使用流程 → 了解如何开始
      ↓
点击 "立即免费开始" → 跳转注册/登录页
      ↓
完成登录 → 自动跳转至 PromptList 页面
      ↓
创建第一个提示词 → 完成引导
```

### 已登录用户访问流程

```
已登录用户访问落地页
      ↓
导航栏显示 "提示词管理" 入口
      ↓
点击 "提示词管理" → 跳转至 /prompts
      ↓
继续使用现有功能
```

### 页面路由策略

| 路由 | 页面类型 | 适用场景 |
|------|----------|----------|
| `/` 或 `/landing` | 落地页 | 新用户、未登录用户默认入口 |
| `/prompts` | 提示词列表 | 已登录用户的主工作区 |
| `/login` | 登录页 | 点击登录按钮跳转 |
| `/prompts/new` | 创建提示词 | 登录后默认跳转目标 |

### 登录成功后的重定向逻辑

```typescript
// 伪代码示例
const handleLoginSuccess = () => {
  // 检查用户是否为首次登录
  if (user.is_first_login) {
    navigate('/prompts/new') // 跳转创建提示词
    markOnboardingCompleted()
  } else {
    // 检查是否有来源页面（从落地页点击登录）
    const fromLanding = sessionStorage.getItem('from_landing')
    if (fromLanding) {
      navigate('/prompts') // 跳转提示词列表
      sessionStorage.removeItem('from_landing')
    } else {
      // 返回之前浏览的页面
      navigate(lastLocation || '/prompts')
    }
  }
}
```

### 功能卡片点击行为

| 功能卡片 | 点击行为 | 目标页面 |
|----------|----------|----------|
| 文件管理 | 未登录：提示登录；已登录：跳转列表 | `/prompts` |
| 版本控制 | 未登录：提示登录；已登录：跳转详情 | `/prompts/:id` |
| 智能优化 | 未登录：提示登录；已登录：跳转详情 | `/prompts/:id` (定位到优化面板) |
| 多模型测试 | 未登录：提示登录；已登录：跳转详情 | `/prompts/:id` (定位到测试面板) |

### CTA 逻辑说明

**关键变更**：去除底部重复的 CTA 按钮，仅在 Hero 区域保留一个主要 CTA。

- **Hero 区域**：包含主 CTA（立即开始使用）和次 CTA（查看演示）
- **底部 CTA**：移除，避免与 Hero CTA 冲突，用户滚动至功能卡片后自然会被引导

## 后端集成策略

### 静态内容管理

落地页是静态页面，不需要后端 API 调用获取内容，但需要以下支持：

| 需求 | 实现方式 | 说明 |
|------|----------|------|
| 国际化文本 | 使用 `frontend/src/i18n/` 翻译文件 | 与主应用共享翻译，保持一致性 |
| 静态资源 | 存放在 `frontend/public/assets/` | Logo、图标等资源使用 CDN 优化 |
| SEO 元数据 | 在 `index.html` 中预渲染 | 或使用 `react-helmet` 动态设置 |

### 缓存策略

| 层级 | 实现方式 | TTL |
|------|----------|-----|
| 浏览器缓存 | `Cache-Control: public, max-age=31536000` | 1 年，适用于静态资源 |
| CDN 缓存 | CDN 边缘缓存 | 24 小时 |
| Service Worker | PWA 离线缓存落地页 | 核心资源优先缓存 |

### 性能监控

| 指标 | 监控工具 | 目标值 |
|------|----------|--------|
| 首屏加载 (LCP) | Vercel Analytics / 自埋点 | < 2.5s |
| 交互延迟 (FID) | Web Vitals API | < 100ms |
| 布局偏移 (CLS) | Web Vitals API | < 0.1 |
| 转化率 | 自定义事件埋点 | 跟踪 CTA 点击、注册完成 |

### 安全考虑

| 安全项 | 实现方式 |
|--------|----------|
| CSP 头 | 设置 `Content-Security-Policy` 限制外部脚本 |
| XSS 防护 | React 自动转义，避免 `dangerouslySetInnerHTML` |
| HTTPS 强制 | 所有资源通过 HTTPS 加载 |
| 隐私合规 | Cookie 同意提示（如需要）

## 多语言支持

### 语言策略

落地页遵循项目的 i18n 架构（参考 `i18n_design.md`）。

| 语言 | 代码 | 文件路径 |
|------|------|----------|
| 中文 | `zh` | `frontend/src/i18n/locales/zh/landing.json` |
| 英文 | `en` | `frontend/src/i18n/locales/en/landing.json` |

### 语言切换行为

1. **语言切换器可见性**：导航栏始终显示语言选择器
2. **切换即时生效**：用户选择语言后，页面内容立即更新，无需刷新
3. **语言持久化**：用户选择的语言保存在 `localStorage`，下次访问自动应用
4. **URL 参数支持**：支持 `?lang=en` 形式，便于分享特定语言版本

| 任务 | 组件/页面 | 验收标准 |
