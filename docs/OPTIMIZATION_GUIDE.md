# 提示词优化功能使用指南

## 📖 简介

InkPrompt 的 AI 优化功能可以帮助你自动改进提示词（Prompt），使其更清晰、更有效、更易于产生高质量的 AI 响应。本指南将详细介绍如何使用这一强大功能。

## ✨ 功能特点

- **智能优化**：基于 GPT 模型的专业提示词工程
- **多场景支持**：针对不同使用场景的定制化优化
- **实时预览**：即时查看优化结果
- **成本透明**：清晰显示 Token 使用和费用
- **一键应用**：轻松替换原有提示词

## 🚀 快速开始

### 步骤 1: 打开提示词编辑器

1. 在提示词列表中，点击任意提示词卡片
2. 进入提示词详情/编辑页面

### 步骤 2: 点击 AI 优化按钮

在编辑器工具栏找到 **"AI 优化"** 按钮（紫色渐变按钮，带闪电图标）：

```
┌─────────────────────────────────────┐
│  提示词编辑器                        │
├─────────────────────────────────────┤
│  [保存]  [取消]  [版本历史]  [✨ AI 优化] │
└─────────────────────────────────────┘
```

### 步骤 3: 选择优化场景

点击 "AI 优化" 按钮后，会弹出场景选择菜单：

```
┌──────────────────────┐
│  选择优化场景         │
├──────────────────────┤
│  ✨ 通用优化          │
│  ✍️  内容创作          │
│  💻 代码生成          │
│  📊 数据分析          │
│  💬 对话交互          │
└──────────────────────┘
```

### 步骤 4: 查看优化结果

- 系统会自动调用 AI 进行优化（通常需要 3-10 秒）
- 优化完成后，内容会自动填充到编辑器
- 右上角会显示优化统计信息：
  ```
  ✅ 优化完成！
  Token: 180 | 成本: $0.0012
  ```

### 步骤 5: 保存或继续编辑

- **满意结果**：点击 "保存" 按钮保存优化后的提示词
- **需要调整**：在编辑器中继续修改内容
- **不满意**：使用 Ctrl+Z (Windows) 或 Cmd+Z (Mac) 撤销

## 💰 成本与性能

### Token 计算

优化功能会消耗 AI Token，包括：
- **输入 Token**：原始提示词 + 优化指令
- **输出 Token**：优化后的提示词

**典型消耗**：
| 原始提示词长度 | 预估 Token | 预估成本 (GPT-3.5) |
|--------------|-----------|-------------------|
| 短 (< 50 字) | 100-200   | $0.0003-0.0006   |
| 中 (50-200 字) | 200-500 | $0.0006-0.0015   |
| 长 (> 200 字) | 500-1000 | $0.0015-0.003    |

### 节省成本的技巧

1. **选择合适的场景**
   - 明确优化目标，避免重复优化
   - 不需要时不要使用 "通用优化"

2. **分步优化**
   - 先简单优化，再针对性调整
   - 避免一次处理过长内容

3. **合理使用频率**
   - 不是每次编辑都需要优化
   - 重大修改时再使用优化功能

4. **选择模型**
   - 默认使用 `gpt-3.5-turbo`（快速、经济）
   - 复杂任务考虑 `gpt-4`（需联系管理员配置）

## 🎓 最佳实践

### ✅ 推荐做法

1. **明确优化目标**
   - 先思考提示词的具体用途
   - 选择最匹配的优化场景

2. **保留原始版本**
   - 优化会自动创建新版本
   - 可通过版本历史回滚

3. **迭代优化**
   - 第一次优化后，可以继续调整
   - 重复优化时可能需要换场景

4. **验证效果**
   - 优化后实际测试提示词效果
   - 根据结果进一步调整

5. **学习改进模式**
   - 观察 AI 如何优化你的提示词
   - 应用这些技巧到未来的提示词创作

### ❌ 避免的做法

1. **盲目优化**
   - 不要对已经很好的提示词反复优化
   - 避免浪费 Token

2. **忽略场景选择**
   - 不要总是用 "通用优化"
   - 场景不匹配会导致不理想的结果

3. **过度依赖**
   - AI 优化是辅助工具，不能完全替代人工
   - 理解优化原理比使用工具更重要

4. **忽略成本**
   - 注意 Token 消耗
   - 避免在测试环境频繁调用

## 🔧 故障排查

### 常见问题

#### 1. 优化按钮无响应

**可能原因**：
- 未登录或 Token 过期
- 网络连接问题
- 提示词内容为空

**解决方案**：
- 刷新页面重新登录
- 检查网络连接
- 确保提示词有内容

#### 2. 优化失败提示

**错误信息**：
```
优化失败，请稍后重试
```

**可能原因**：
- OpenAI API 配额用尽
- API Key 无效
- 服务器临时故障

**解决方案**：
- 等待几分钟后重试
- 联系管理员检查 API 配置
- 查看系统状态公告

#### 3. 优化结果不理想

**可能原因**：
- 场景选择不合适
- 原始提示词过于简单或复杂
- AI 模型理解偏差

**解决方案**：
- 尝试不同的优化场景
- 手动调整部分内容后再优化
- 提供更多上下文信息
- 使用版本历史回滚后重试

#### 4. 优化速度慢

**正常情况**：
- 简单提示词：3-5 秒
- 复杂提示词：5-10 秒
- 服务器繁忙时：10-15 秒

**异常情况**：
- 超过 30 秒无响应
- 可能是网络或服务器问题
- 刷新页面重试

### 自定义优化要求

虽然当前界面不支持自定义要求（计划中的功能），但你可以：

1. **在提示词中包含优化提示**
   ```
   [请保持专业术语]

   ```

2. **分步优化**
   - 第一次：选择基础场景优化
   - 第二次：手动调整后再优化

3. **组合使用**
   - 先用 "通用优化" 改善结构
   - 再用特定场景优化细节

### 优化模式识别

观察 AI 的优化模式，学习提示词工程：

| 优化模式 | 说明 | 示例 |
|---------|------|------|
| **结构化** | 添加编号列表、分段 | 1. 2. 3. ... |
| **具体化** | 替换模糊词汇为具体描述 | "好的文章" → "1500字的深度分析文章" |
| **上下文化** | 添加背景信息和约束 | 指定目标受众、输出格式 |
| **示例化** | 提供输入输出示例 | "输入：... 输出：..." |
| **角色化** | 定义 AI 的角色和专业领域 | "你是专业的..." |

### 与其他功能结合

1. **版本历史 + 优化**
   - 优化会自动创建版本
   - 可对比优化前后效果
   - 随时回滚到之前的版本

2. **标签 + 优化**
   - 给优化后的提示词打标签
   - 分类管理不同场景的提示词
   - 快速找到特定类型的优化案例

3. **搜索 + 优化**
   - 搜索相似的已优化提示词
   - 学习优秀的提示词模式
   - 建立个人提示词库

### 案例 2: 代码生成优化

**场景**：API 端点实现

**优化前** (28 字)：
```
写一个用户注册的 API 接口，用 FastAPI
```

**优化后** (412 字)：
```
使用 FastAPI 实现用户注册 API 端点：

功能需求：
- 端点：POST /api/auth/register
- 框架：FastAPI 0.100+
- 数据库：SQLAlchemy ORM + PostgreSQL

请求体 Schema：
{
  "email": "user@example.com",
  "password": "SecurePass123!",
  "username": "johndoe"
}

业务逻辑：
1. 验证邮箱格式（使用 pydantic EmailStr）
2. 检查邮箱/用户名是否已存在
3. 密码强度验证（至少8位，含大小写字母和数字）
4. 密码加密存储（使用 bcrypt）
5. 创建用户记录
6. 返回 JWT Token

响应格式：
- 成功（201）: { "user_id": "uuid", "access_token": "..." }
- 失败（400）: { "detail": "Email already registered" }

技术要求：
1. 使用 Pydantic models 进行数据验证
2. 异步函数（async def）
3. 错误处理（HTTPException）
4. 包含类型提示
5. 添加 API 文档字符串

安全考虑：
- SQL 注入防护（ORM 自动处理）
- 密码不记录日志
- 限制注册频率（使用 slowapi，可选）

请提供完整的实现代码，包括：
- Pydantic schemas
- API 路由函数
- 数据库模型（如需要）
- 基本单元测试
```

**效果**：Token 从 28 → 412，技术规格清晰，可直接实现

6. 可导出为 PDF 报告

分析深度：
- 不仅展示数据，还要挖掘洞察
- 识别异常和趋势
- 提供可执行的业务建议

请提供分析框架代码，包括数据加载、清洗、分析和可视化的完整实现。
