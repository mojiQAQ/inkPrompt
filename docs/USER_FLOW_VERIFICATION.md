### 6.1 优化按钮显示

- [ ] 编辑已保存提示词时**显示**优化按钮
- [ ] 按钮位于内容输入框右上角
- [ ] 按钮显示为紫色渐变带闪电图标

### 6.2 场景选择
- [ ] 点击优化按钮显示场景下拉菜单
- [ ] 显示 5 个优化场景：
  - [ ] ✨ 通用优化
  - [ ] ✍️ 内容创作
  - [ ] 💻 代码生成
  - [ ] 📊 数据分析
  - [ ] 💬 对话交互
- [ ] 每个场景有图标和名称

### 6.3 优化流程
- [ ] 选择一个场景
- [ ] 按钮变为"优化中..."状态
- [ ] 等待 3-10 秒（取决于内容长度）
- [ ] 优化成功后：
  - [ ] 显示成功 Toast 提示
  - [ ] Toast 中显示 Token 数量
  - [ ] Toast 中显示估算成本
  - [ ] 内容输入框中的内容已自动更新为优化后的版本
- [ ] 优化失败时：
  - [ ] 显示错误 Toast 提示
  - [ ] 内容不变

### 6.4 优化结果处理
- [ ] 查看优化后的内容
- [ ] 满意结果 → 点击"保存更改"
- [ ] 不满意结果 → 按 Ctrl+Z (或 Cmd+Z) 撤销
- [ ] 可以重新选择其他场景再次优化
- [ ] 保存时自动创建新版本

**测试场景**：
```
1. 创建提示词，内容：
   "帮我写一篇关于人工智能的文章要包括AI的定义发展历史和未来趋势"

2. 使用"通用优化"场景
   → 内容应变得更结构化、清晰

3. 使用"内容创作"场景
   → 内容应更富有创意表达

4. 使用"代码生成"场景（不太匹配）
   → 仍能正常优化，但效果可能不如其他场景

5. 测试超长文本（>1000字）
   → 仍能正常优化，但时间较长，成本较高
```

- [ ] 每次优化后 Toast 显示估算成本（美元）

## 🎯 15. 核心流程端到端测试

### 场景 1：新用户首次使用
```
1. 访问应用 → 跳转到登录页
2. 使用 Google 登录 → 成功登录
3. 查看空的提示词列表
4. 点击"新建提示词"
5. 输入名称："Python 代码审查助手"
6. 输入内容："请帮我审查以下 Python 代码..."
7. 添加标签："python", "code"
8. 创建提示词 → 成功
9. 提示词出现在列表中
```

### 场景 2：编辑和优化提示词
```
1. 点击提示词进入编辑页
2. 查看版本历史 → 显示 v1
3. 点击"AI 优化"按钮
4. 选择"代码生成"场景
5. 等待优化完成
6. 查看优化后的内容
7. 满意 → 点击"保存更改"
8. 填写版本说明："AI 优化后的版本"
9. 保存成功
10. 版本历史显示 v2
```

### 场景 3：搜索和筛选
```
1. 创建多个提示词（不同标签和内容）
2. 在搜索框输入"python"
3. 查看搜索结果 → 只显示包含 python 的提示词

5. 添加标签筛选："code"
6. 结果进一步筛选
7. 切换排序：按 Token 数降序
8. 清除所有筛选 → 恢复所有提示词
```

### 场景 4：版本回滚

```
1. 编辑提示词，修改内容 3 次
2. 版本历史显示 v1, v2, v3, v4

4. 确认恢复
5. 内容恢复到 v1 的状态
6. 版本历史显示新版本 v5（内容同 v1）
```
